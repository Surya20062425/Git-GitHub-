Here are comprehensive resources to learn **Git** (the version control system) and **GitHub** (the hosting platform), organized by type and skill level.

---

## Official Documentation (Start Here)

| Resource | What It Covers | Best For |
|----------|---------------|----------|
| **[Git Official Docs](https://git-scm.com/doc)** | Complete Git reference, Pro Git book (free), videos | Reference and deep dives |
| **[GitHub Docs](https://docs.github.com/en)** | Repositories, pull requests, actions, pages, codespaces | Platform-specific workflows |
| **[GitHub Skills](https://skills.github.com/)** | Interactive courses (Introduction to GitHub, CI/CD, Pages) | Hands-on beginners |

---

## Interactive Learning & Tutorials

- **[Learn Git Branching](https://learngitbranching.js.org/)** — Visual, interactive tutorial for understanding branching, merging, and rebasing. Highly recommended.
- **[Oh My Git!](https://ohmygit.org/)** — A free, open-source game that teaches Git concepts interactively.
- **[GitHub Learning Lab](https://lab.github.com/)** — Step-by-step bot-guided projects inside real repositories.
- **[Codecademy: Learn Git & GitHub](https://www.codecademy.com/learn/learn-git)** — Structured course with exercises.
- **[freeCodeCamp: Git & GitHub Course](https://www.freecodecamp.org/news/git-and-github-for-beginners/)** — Free, comprehensive written + video guides.

## Books (Free)

- **[Pro Git](https://git-scm.com/book/en/v2)** by Scott Chacon & Ben Straub — The definitive free book. Covers everything from basics to internals.
- **[GitHub Actions Essentials](https://www.gitbook.com/book/sdras/nerds-pages)** by Sarah Drasner — Focused on CI/CD with GitHub Actions.

## Practice & Playgrounds

- **[GitHub](https://github.com/)** itself — Create a repo, push code, open issues, try pull requests.
- **[GitHub Codespaces](https://github.com/features/codespaces)** — Cloud dev environment; practice without local setup.
- **[First Contributions](https://github.com/firstcontributions/first-contributions)** — A guided project to make your first open-source contribution.

---

## Key Concepts to Master (Learning Path)

1. **Git Basics**: `init`, `clone`, `add`, `commit`, `status`, `log`, `diff`
2. **Branching & Merging**: `branch`, `checkout`, `merge`, merge conflicts
3. **Remote Workflows**: `remote`, `push`, `pull`, `fetch`
4. **Undoing Changes**: `revert`, `reset`, `stash`, `clean`
5. **History Rewriting**: `rebase` (interactive), `cherry-pick`, `amend`
6. **GitHub Workflow**: Forking, pull requests, code review, issues, projects
7. **Automation**: GitHub Actions (CI/CD), webhooks
8. **Collaboration**: Branch protection rules, CODEOWNERS, semantic versioning

---

**Tip**: Start with **Learn Git Branching** (interactive) + **GitHub Skills** (hands-on), then use **Pro Git** for anything you want to understand deeply. Practice by uploading a personal project to GitHub and using branches for features.
